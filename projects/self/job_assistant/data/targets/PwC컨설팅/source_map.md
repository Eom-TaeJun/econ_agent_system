## 0. 이 직무의 핵심 포지셔닝

PwC RA는 "클라이언트가 보고 싶은 숫자"가 아닌 "실제로 의미 있는 숫자"를 찾는 자리.
분석 기술 + 비즈니스 판단력을 동시에 요구.
전형: 서류(Resume) → Fit 인터뷰(유선/Zoom) → 본사 대면(1:2, 1.5시간)

## 2. 항목별 소재 설계

### A. 지원 동기 (간결하게)

**핵심 메시지**: 논문에서 "Fact보다 Insight가 핵심"임을 체감 → PwC의 Community of solvers 비전과 연결

**소재**:
- 연준 통화정책 논문: 변수 값이 아닌 "시장의 정보처리 방식 자체가 변화했다"는 구조적 변화를 발견
- 이 경험에서 데이터 분석의 목적이 단순 수치 확인이 아닌 의사결정에 필요한 인사이트 도출임을 확인
- 컨설팅 RA = 분석력과 판단력을 실전에 동시에 검증할 수 있는 자리

**핵심 문장 초안**:
- "논문을 쓰는 과정에서 데이터가 무엇을 말하는지보다 어떻게 해석할 것인가가 핵심임을 배웠습니다."
- "PwC가 추구하는 Community of solvers처럼 분석 결과를 실제 문제 해결로 연결하고 싶습니다."

---

### B. 직무 역량 1 — 데이터 분석 설계력 (논문)

**매핑 강도: ★★★★★**

**소재**:
- 50개 거시경제 변수 → LASSO로 유의미한 변수 선택 (Signal vs Noise 분리)
- Post-LASSO OLS + Newey-West HAC 3단계 체계 독립 설계
- 68회 롤링 추정으로 초기 유망 결과 재검증 → 안정성 확인 습관
- Chow Test로 구조 변화 발견 → "변수 값이 아닌 시장의 정보처리 방식 자체가 변화"

**컨설팅 프레이밍**:
- LASSO = 복잡한 고객 데이터에서 핵심 지표 추출하는 전처리 설계력
- 68회 롤링 = 클라이언트 보고 전 결론 안정성 확인하는 치밀함
- Chow Test = 표면 현상 이면의 근본 원인을 짚어내는 분석 태도

**핵심 문장 초안**:
- "50개 변수에서 LASSO로 핵심 지표를 선별하고, 68회 롤링 추정으로 결과의 안정성을 검증했습니다."
- "좋아 보이는 결과를 그대로 보고하지 않고 구조적 변화 여부를 Chow Test로 확인하는 습관을 갖추었습니다."

---

### C. 직무 역량 2 — End-to-End 파이프라인 (MLB Stats)

**매핑 강도: ★★★★**

**소재**:
- API 연동 + 캐싱 + SQLite 레포지터리 패턴으로 데이터 수집·저장 구축
- wOBA, wRC+, FIP 등 7개 이상 도메인 지표 직접 정의·구현
- 인터랙티브 대시보드 + PDF/HTML 자동 생성 (4,940+ 라인, 180 테스트)
- Python 코드를 유지보수 가능한 구조로 작성

**컨설팅 프레이밍**:
- PwC 글로벌 리테일 데이터 분석 = 데이터 수집 → 전처리 → 지표 정의 → 시각화 → PPT 보고의 흐름
- 이 전체 사이클을 혼자 완성한 경험이 있음 → 즉시 투입 가능

**핵심 문장 초안**:
- "데이터 수집부터 지표 정의, PDF 리포트 자동 생성까지 end-to-end 파이프라인을 직접 구축했습니다."
- "일회성 분석 코드가 아닌 유지보수 가능한 구조로 작성하는 습관을 갖고 있습니다."

---

### D. 팀 협업 & 도메인 적응력

**소재 1 — AI 프로젝트 설득 (협업)**:
- 팀 내 설계 방향 불일치 발생 → 말 대신 작동하는 프로토타입으로 설득
- AI 환각 문제 사전 인식 → 스코프 현실적으로 조정

**소재 2 — POSCO 철강 불량 예측 (도메인 러닝)**:
- 사전 지식 없는 도메인 → 공정 원리부터 학습 후 분석 구조 설계
- "단순 예측"이 아닌 "실제 개선안"으로 문제 재정의
- 글로벌 리테일 = 철강만큼 생소한 도메인일 수 있음 → "처음 보는 도메인에도 투입 가능" 증명

**핵심 문장 초안**:
- "설계 방향이 충돌했을 때 작동하는 프로토타입을 직접 만들어 팀을 설득했습니다."
- "사전 지식 없는 철강 공정 도메인에서 공정 원리부터 학습해 분석 구조를 설계한 경험이 있습니다."

## 3. 면접 대비 핵심 답변 방향

| 질문 | 내 소재 | 한 줄 요약 |
|------|---------|-----------|
| 지원 이유 | 논문 → Insight 체험 | Fact보다 해석이 핵심임을 체감, 컨설팅에서 검증하고 싶음 |
| 문제해결 사례 | LASSO + 68회 롤링 | 유망한 결과에 안주하지 않고 안정성 재검증 |
| 팀 역할 | AI 프로토타입 설득 | 말이 아닌 작동하는 산출물로 방향 전환 성공 |
| 분석 기법 | Post-LASSO OLS, wOBA 등 지표 설계 | 방법론을 문제에 맞게 직접 설계한 경험 |
| 기여 계획 | End-to-End 파이프라인 경험 | 데이터 수집→시각화→보고 전 사이클 즉시 투입 가능 |

## 4. 사용하지 않을 소재

- 야구 서사 (컨설팅 맥락에서 약함)
- 경진대회 순위 (성과 수치가 없으면 활용도 낮음)
- eco_system 튜닝 (PwC는 리테일 도메인이므로 직접 연결 어려움)
