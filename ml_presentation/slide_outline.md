# 슬라이드 구성표 (확정)

> 총 슬라이드 예상: 1~5차시 51장 + 6차시 13장 = 64장
> 부록: PCA / 군집화 / AIC·BIC (별도 파일)

---

## 1차시 (8장) — 왜 ML인가

| 장 | 제목 | 핵심 비주얼 | 원본 |
|---|---|---|---|
| 1 | 오늘 AI가 내린 결정들 | 일상 앱 3개 아이콘 + "공통점은?" | 슬5 앞 |
| 2 | 데이터가 폭발했다 | 데이터 증가 + 연산력 타임라인 | 슬5 앞 |
| 3 | 같은 데이터, 다른 질문 | 통계학(왜?) ↔ ML(얼마나?) 좌우 비교 | 슬1 |
| 4 | ML이 허용한 것: 블랙박스 | "답만 맞으면 되는 거 아닌가요?" | 슬4 |
| 5 | ML이 답하는 질문 지도 ⭐ | 회귀/분류/군집/해석 4분류 다이어그램 | 신규 |
| 6 | 실제로 어디에 쓰이나 | 금융·마케팅·의료·스포츠 4칸 카드 | 신규 |
| 7 | **[추가] LLM 시대에도 ML인가?** | GPT 아이콘 ↔ 테이블 데이터 → "LLM 내부도 ML" | 신규 |
| 7-1 | **[추가] 2026년: ML이 AI 시스템의 부품이 된다** | 에이전틱 AI 구조도 (사용자→에이전트→ML 예측 모듈→도구) | 신규 |
| 8 | 예고: 자유에는 대가가 따른다 | 다음 차시 연결 한 줄 | 슬5 뒤 |

## 2차시 (12장) — 자유에는 대가가 따른다

| 장 | 제목 | 핵심 비주얼 | 원본 |
|---|---|---|---|
| 1 | 출발점: 선형회귀와 다중공선성 | 수식 시각화 + "계산 거부" 상황 | 슬2+3 압축 |
| 2 | ML의 대답: 결과만 맞으면 | 블랙박스 허용 전환 | 슬4 |
| 3 | 자유의 대가: 과적합 ⭐ | 기출문제 외우기 학생 비유 | 슬6 |
| 4 | 규제화 등장 | 오차 + β 크기 동시 제약 | 슬7 |
| 5 | Ridge vs Lasso: 언제 뭘 쓰나 | 억제(Ridge) vs 제거(Lasso) 비교 | 슬8+9 |
| 6 | 규제화의 진짜 의미 | Bias 희생 → Variance 감소 | 슬10 |
| 6-1 | **[추가] Confusion Matrix: 지표의 출발점** | TP/FP/FN/TN 2×2 표 + Precision·Recall 수식 위치 표시 | 신규 |
| 6-2 | **[추가] Train / Validation / Test: 왜 세 개로 나누나** | 삼분할 다이어그램 + "Test는 딱 한 번만" | 신규 |
| 7 | 분류 문제 맛보기 + 교차검증 | 회귀→분류 전환 + 정확도 함정 + K-Fold 5분할 다이어그램 | 슬11~12-1 압축+보강 |
| 8 | **[추가] 실무 함정: Data Leakage** | 타임라인 그래프 + 미래 데이터 침투 경고 박스 | 신규 |
| 7-1 | **[추가] 어떤 지표로 모델을 평가하나** | 상황별 지표 선택 의사결정표 | 신규 |
| 9 | 예고: 직선으로 안 되는 세상 | 다음 차시 연결 | 슬12-2 |

## 3차시 (8장) — 나무로 규칙을 만든다

| 장 | 제목 | 핵심 비주얼 | 원본 |
|---|---|---|---|
| 1 | 직선으로 설명 안 되는 세상 | 비선형 곡선 그래프 | 슬13 예고 |
| 2 | 스무고개: 의사결정나무 | IF-THEN 트리 다이어그램 | 슬13+14 |
| 3 | 나무의 장점과 한계 | 장점 3카드 + 불안정성 문제 | 슬15+16 |
| 4 | Bias-Variance Tradeoff ⭐ | 2×2 매트릭스 다이어그램 | 슬17 |
| 5 | 해결책 1: 집단지성 (Bagging) | 병렬 학습 플로우 | 슬18 |
| 6 | 랜덤 포레스트: 숲을 이루다 | 나무 → 숲 이미지 | 슬19 |
| 7 | **[추가] 실사례: 나무가 실무에서 하는 일** | DT → 대출 심사 규칙 / RF → 신용카드 이상거래 2칸 카드 | 신규 |

## 6차시 (13장) — EDA + 과제 (보강 세션)

### EDA 강의 (9장)

| 장 | 제목 | 핵심 비주얼 |
|---|---|---|
| 1 | 왜 EDA가 모델보다 먼저인가 | "쓰레기 IN → 쓰레기 OUT" |
| 2 | 데이터 진단 체크리스트 | 결측/이상치/중복/타입 4항목 |
| 3 | 결측치: 얼마나? 어디에? | 결측 히트맵 + 비율별 처리 기준 (5%/30% 임계) |
| 4 | 이상치: 에러인가, 사건인가? | 박스플롯 + 도메인 확인 → 제거/유지 의사결정 흐름 |
| 5 | Before/After 비교가 전부다 ⭐ | 같은 축 / 같은 단위 / 숫자 증명 |
| 6 | 관계 보기 | 상관 히트맵 + 산점도 |
| 7 | 파생 변수: 숨겨진 패턴 | 시간대·요일·계절 예시 |
| 8 | 흔한 실수 4가지 | 레드 경고 스타일 카드 |
| 9 | EDA → ML 연결고리 | 전처리 완료 → 모델 → 의미 있는 결과 |
